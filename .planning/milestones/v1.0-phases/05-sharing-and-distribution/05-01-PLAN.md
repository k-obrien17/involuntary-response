  <name>Task 1: OG meta tags via Vercel serverless function</name>
  <files>client/index.html, client/api/og.js, client/vercel.json</files>
  <action>
1. Update `client/index.html` `<head>` to add OG placeholder meta tags after the existing `<meta name="description">` tag:
   - `<meta property="og:title" content="__OG_TITLE__" />`
   - `<meta property="og:description" content="__OG_DESCRIPTION__" />`
   - `<meta property="og:image" content="__OG_IMAGE__" />`
   - `<meta property="og:type" content="article" />`
   - `<meta property="og:url" content="__OG_URL__" />`
   - `<meta name="twitter:card" content="summary_large_image" />`
   - `<meta name="twitter:title" content="__OG_TITLE__" />`
   - `<meta name="twitter:description" content="__OG_DESCRIPTION__" />`
   - `<meta name="twitter:image" content="__OG_IMAGE__" />`
   - Also add RSS autodiscovery: `<link rel="alternate" type="application/rss+xml" title="Involuntary Response" href="/api/feed" />`

2. Create `client/api/og.js` (Vercel serverless function):
   - Read `dist/index.html` once (cache in module scope for warm starts)
   - Extract slug from `req.url` by stripping `/posts/` prefix
   - Fetch post data from `RENDER_API_URL/api/posts/{slug}` with 3-second timeout via `AbortSignal.timeout(3000)`
   - On success: build title as `"{author.displayName} on {embed.title} | Involuntary Response"` (if embed has title) or `"{author.displayName} | Involuntary Response"` (fallback), description as `post.body.slice(0, 200)`, image as `post.embed?.thumbnailUrl` falling back to `${SITE_URL}/og-default.png`
   - On fetch failure or non-200: use defaults (title: "Involuntary Response", description: "Short-form music takes from people who care about music.", image: default OG image)
   - Replace `__OG_TITLE__`, `__OG_DESCRIPTION__`, `__OG_IMAGE__`, `__OG_URL__` placeholders in HTML using regex `.replace()` with global flag
   - Escape HTML entities in title/description (`& " < >`) to prevent XSS in meta tags
   - Set response headers: `Content-Type: text/html; charset=utf-8`, `Cache-Control: public, s-maxage=3600, stale-while-revalidate=86400`
   - Use `export default async function handler(req, res)` signature (Vercel serverless convention)
   - Environment variables: `SITE_URL` (defaults to 'https://involuntary-response.vercel.app'), `RENDER_API_URL` (no default, graceful degradation if missing)

3. Create a simple default OG image placeholder at `client/public/og-default.png`. Use a minimal approach: create a simple 1200x630 solid dark image with site name. Since we cannot generate images programmatically, create a small HTML file at `client/public/og-default.html` as documentation, and for now use a 1x1 transparent PNG as a placeholder (the user will replace it). Actually -- just note in the plan summary that the user needs to provide `client/public/og-default.png` (1200x630). Do NOT create a binary image file.

4. Update `client/vercel.json` to add the OG rewrite BEFORE the catch-all:
   ```json
   {
     "rewrites": [
       { "source": "/posts/:slug", "destination": "/api/og" },
       { "source": "/(.*)", "destination": "/" }
     ]
   }
   ```
   Order matters: specific `/posts/:slug` must come before the catch-all `/(.*)`

CRITICAL: All OG image URLs must be absolute (include SITE_URL prefix). The `thumbnailUrl` from Spotify oEmbed is already absolute. The default image must also be absolute (`${SITE_URL}/og-default.png`).

AVOID: Do not use react-helmet or any client-side approach -- social crawlers do not execute JavaScript.

<task type="auto">
  <name>Task 2: RSS feed Express endpoint</name>
  <files>server/routes/feed.js, server/index.js, server/package.json</files>
  <action>
1. Install the `feed` npm package: `cd /Users/keithobrien/server && npm install feed`

2. Create `server/routes/feed.js`:
   - Import `Router` from express, `Feed` from 'feed', and `db` from '../db/index.js'
   - Single GET `/` route handler
   - Construct `siteUrl` from `process.env.FRONTEND_URL || 'http://localhost:5173'`
   - Create `new Feed({ title: 'Involuntary Response', description: 'Short-form music takes from people who care about music.', id: siteUrl, link: siteUrl, language: 'en', copyright: '${new Date().getFullYear()} Involuntary Response' })`
   - Query recent posts: `SELECT p.id, p.slug, p.body, p.created_at, u.display_name AS author_display_name FROM posts p JOIN users u ON p.author_id = u.id ORDER BY p.created_at DESC LIMIT 20`
   - Batch-fetch embeds for those posts using `IN(${placeholders})` pattern (same pattern used in browse.js and feed endpoint): `SELECT post_id, title, original_url FROM post_embeds WHERE post_id IN (...)` with spread params
   - Build embedMap keyed by post_id
   - For each post, call `feed.addItem()` with:
     - `title`: `"{author_display_name} on {embed.title}"` if embed exists with title, else `"Post by {author_display_name}"`
     - `id`: `${siteUrl}/posts/${post.slug}`
     - `link`: `${siteUrl}/posts/${post.slug}`
     - `description`: `post.body.slice(0, 200)`
     - `content`: `<p>${post.body}</p>` plus `<p><a href="${embed.original_url}">${embed.title || 'Listen'}</a></p>` if embed exists
     - `date`: `new Date(post.created_at)`
     - `author`: `[{ name: post.author_display_name }]`
   - Set response headers: `Content-Type: application/rss+xml; charset=utf-8`, `Cache-Control: public, max-age=3600`
   - Return `feed.rss2()`
   - Wrap in try/catch, log errors, return 500 on failure
   - Export `router` as default

3. Mount in `server/index.js`:
   - Add import: `import feedRoutes from './routes/feed.js';`
   - Add mount BEFORE the health check: `app.use('/api/feed', feedRoutes);`

IMPORTANT: The feed links MUST use `process.env.FRONTEND_URL` (the Vercel frontend domain), NOT the Render API domain. RSS items should link to the website, not the API.

AVOID: Do not hand-roll XML. The `feed` package handles RSS 2.0 spec compliance, date formatting (RFC-822), and XML escaping.
  </action>
