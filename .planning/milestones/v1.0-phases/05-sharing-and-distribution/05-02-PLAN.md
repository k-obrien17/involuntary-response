**Part A: Theme Infrastructure (do these first)**

1. Create `client/src/context/ThemeContext.jsx`:
   - Export `ThemeProvider` component and `useTheme` hook
   - State: `preference` initialized from `localStorage.getItem('theme') || 'system'`
   - Helper `getEffectiveTheme(stored)`: returns 'dark' if stored==='dark', 'light' if stored==='light', otherwise checks `window.matchMedia('(prefers-color-scheme: dark)').matches`
   - Helper `applyTheme(theme)`: `document.documentElement.classList.toggle('dark', theme === 'dark')`
   - useEffect on `preference` change: call `applyTheme(getEffectiveTheme(preference))`
   - useEffect for system mode: when `preference === 'system'`, listen to `matchMedia('(prefers-color-scheme: dark)')` change events to re-apply theme. Cleanup listener on unmount/mode change.
   - `setTheme(value)` function: if value==='system', remove 'theme' from localStorage; otherwise `localStorage.setItem('theme', value)`. Update preference state.
   - Context value: `{ preference, setTheme }`

2. Add FOUC prevention script to `client/index.html` `<head>`, BEFORE the closing `</head>` tag (and before any `<script>` tags that load React):
   ```html
   <script>
     (function() {
       var t = localStorage.getItem('theme');
       if (t === 'dark' || (!t && window.matchMedia('(prefers-color-scheme: dark)').matches)) {
         document.documentElement.classList.add('dark');
       }
     })();
   </script>
   ```
   NOTE: Plan 05-01 also modifies index.html to add OG meta tags. These changes are in different sections (OG tags in `<head>` meta area, FOUC script at end of `<head>`), so there should be no merge conflict.
