  <action>
    Visual and functional verification of Phase 3 implementation. No code changes -- human confirms the reading experience works correctly.

    Start both servers if not running:
    - `cd server && npm run dev` (port 3001)
    - `cd client && npm run dev` (port 5173)

    **Feed (DISC-01):**
    1. Visit http://localhost:5173/ -- you should see a reverse-chronological feed of posts (newest first)
    2. If no posts exist yet, log in and create 2-3 test posts at /posts/new (with and without embeds + tags) and return to the home page
    3. Verify each post shows: large-type body text, tags as #hashtags, relative time ("X minutes ago"), and author display name
    4. If any post has an embed, verify it shows a click-to-load placeholder (play button + provider name), NOT an active iframe
    5. Click the embed placeholder -- verify the Spotify or Apple Music iframe loads in place

    **Permalink (SHAR-01):**
    6. Click the relative time link on any post -- verify it navigates to /posts/:slug
    7. On the permalink page, verify: larger body text than the feed, full date (e.g., "February 27, 2026"), author name, and tags
    8. If the post has an embed, verify it shows the full iframe directly (not a placeholder)
    9. If you are the post author, verify the "Edit" link appears

    **Typography (DSGN-01):**
    10. On both the feed and permalink page, verify body text is large and readable with generous whitespace
    11. Content should be in a single, centered column -- not stretching to full viewport width

    **Responsive (DSGN-02):**
    12. Resize the browser to mobile width (~375px) -- verify the layout remains single-column, text is readable, and nothing overflows horizontally
    13. Verify the "Older posts" button appears if you have more than 20 posts (or test with `?limit=1` on the API)
  </action>

  <done>Feed renders correctly, embeds work as click-to-load, permalinks load, text-first layout confirmed on desktop and mobile</done>
