## Goal

Make `authenticateToken` and `optionalAuth` middleware read user role/status from the database on every request instead of trusting JWT payload, add `GET /auth/me` endpoint, and add rate limiter to `POST /auth/reset-password`.

## Context

**Current state:** `authenticateToken` already does a DB lookup for `is_active` but still uses `decoded` (JWT payload) as `req.user`, so role is stale. `optionalAuth` does no DB check at all. No `/auth/me` endpoint exists. `POST /auth/reset-password` has no rate limiter.

**User decisions (LOCKED):**
- `authenticateToken` DB failure -> 503 Service Unavailable
- `optionalAuth` DB failure -> treat as unauthenticated (graceful degradation)
- Deactivated users get generic 401 (don't reveal account status)
- DB errors logged with user ID for debugging
- `/auth/me` returns same shape as login: `{ id, username, displayName, role, email }`
- No token refresh, just validate and return current user data
- Password reset rate limiter matches `forgotPasswordLimiter` config

**Claude's discretion:** Whether to extract shared auth helper, exact DB query structure.

## Tasks

<task id="15-01-01">
<title>Harden authenticateToken to use DB role</title>
<file>server/middleware/auth.js</file>

Modify `authenticateToken`:
1. After `jwt.verify(token, JWT_SECRET)`, run: `SELECT id, email, role, username, display_name, is_active FROM users WHERE id = ?` using `decoded.id`
2. If DB query throws, return `503 Service Unavailable` with `{ error: 'Service temporarily unavailable' }`. Log the error with `console.error('Auth DB error for user', decoded.id, err)`
3. If no user found OR `user.is_active === 0`, return `401` with `{ error: 'Authentication required' }` (generic, don't reveal deactivation)
4. Set `req.user` to `{ id: user.id, email: user.email, role: user.role, username: user.username, displayName: user.display_name }` — sourced from DB, not JWT
5. The JWT is only used to extract the user ID for the DB lookup

**Key:** The catch block must distinguish between JWT verification errors (return 403) and DB errors (return 503). Use separate try/catch blocks: outer try for jwt.verify, inner try for DB.

**Verify:** `req.user.role` now comes from the database, not from the JWT payload.
</task>

<task id="15-01-02">
<title>Harden optionalAuth to check DB</title>
<file>server/middleware/auth.js</file>

Modify `optionalAuth` to be async and do a DB lookup:
1. After `jwt.verify(token, JWT_SECRET)`, run the same DB query as authenticateToken
2. If user found and `is_active !== 0`, set `req.user` from DB data (same shape as authenticateToken)
3. If user not found, is_active === 0, or DB query fails -> do NOT set `req.user`, just call `next()` (treat as unauthenticated). Log DB errors: `console.error('optionalAuth DB error for user', decoded.id, err)`
4. JWT verify failure -> treat as unauthenticated (same as current behavior)

**Key:** `optionalAuth` must now be `async`. All existing route registrations already use it as middleware — Express 5 handles async middleware natively, so no wrapper needed.

**Verify:** A deactivated user with a valid JWT is treated as unauthenticated (no `req.user`) in optional-auth routes.
</task>

<task id="15-01-03">
<title>Add GET /auth/me endpoint and reset-password rate limiter</title>
<file>server/routes/auth.js</file>

1. Import `authenticateToken` from `../middleware/auth.js`
2. Add `GET /auth/me` route with `authenticateToken` middleware:
   - On success (middleware passed), return `res.json({ user: { id: req.user.id, email: req.user.email, displayName: req.user.displayName, username: req.user.username, role: req.user.role } })`
   - No rate limiter beyond global (per user decision)
3. Add rate limiter to `POST /api/auth/reset-password`:
   - Create `resetPasswordLimiter` matching `forgotPasswordLimiter` config: `windowMs: 15 * 60 * 1000, max: 5, message: { error: 'Too many attempts, try again later' }`
   - Apply to the existing reset-password route

**Verify:** `GET /auth/me` with a valid token returns current user data from DB. `POST /auth/reset-password` returns 429 after 5 attempts in 15 minutes.
</task>
