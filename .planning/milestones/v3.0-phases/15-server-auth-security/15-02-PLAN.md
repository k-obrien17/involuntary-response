Add helmet security headers, global rate limiter, RSS feed XSS fix, displayName sanitization, and express.json body size limit.

## Context

**Current state:** No security headers. No global rate limiter (only per-endpoint). RSS feed interpolates `embed.originalUrl` and `embed.title` directly into HTML content. No displayName sanitization on registration. No body size limit on `express.json()`.

- X-Frame-Options: DENY
- HSTS enabled
- All standard helmet headers enabled
- Global rate limiter: 200 req/min per IP, applied before specific limiters
- Include Retry-After header in 429 responses

**Claude's discretion:** Exact helmet config object, express.json limit (100kb suggested), displayName sanitization approach, RSS XSS fix approach.

<task id="15-02-01">
<title>Add helmet security headers and global rate limiter</title>
<file>server/index.js</file>
<file>server/package.json</file>

1. Install `helmet` package: add to server/package.json dependencies
2. In `server/index.js`, import `helmet` and `rateLimit` from `express-rate-limit`
3. Add helmet middleware BEFORE cors, configured with:
   - `contentSecurityPolicy` with `directives`:
     - `defaultSrc: ["'self'"]`
     - `frameSrc: ["'self'", "https://open.spotify.com", "https://embed.music.apple.com", "https://www.youtube.com", "https://w.soundcloud.com", "https://bandcamp.com", "https://*.bandcamp.com"]`
     - `imgSrc: ["'self'", "https:", "data:"]` (needed for Gravatar and external images)
     - `scriptSrc: ["'self'"]`
     - `styleSrc: ["'self'", "'unsafe-inline'"]` (Tailwind uses inline styles)
   - `frameguard: { action: 'deny' }` (X-Frame-Options: DENY)
   - `hsts: { maxAge: 31536000, includeSubDomains: true }` (1 year)
4. Add global rate limiter AFTER cors, BEFORE routes:
   - `windowMs: 60 * 1000` (1 minute)
   - `max: 200`
   - `standardHeaders: true` (includes Retry-After in 429 responses)
   - `legacyHeaders: false`
   - `message: { error: 'Too many requests, please try again later' }`
5. Add `{ limit: '100kb' }` to `express.json()` call

**Middleware order in index.js:**
1. `helmet(...)`
2. `cors(...)`
3. `globalLimiter`
4. `express.json({ limit: '100kb' })`
5. Routes

**Verify:** Response headers include `Content-Security-Policy`, `Strict-Transport-Security`, `X-Frame-Options: DENY`, `X-Content-Type-Options: nosniff`.
</task>

**RSS XSS fix (SEC-02):**

In `server/routes/feed.js`, HTML-entity-encode `embed.originalUrl` and `embed.title` before interpolating into content string:

1. Add a simple `escapeHtml` helper at the top of the file:
   ```
   function escapeHtml(str) {
     if (!str) return '';
     return str.replace(/&/g, '&amp;').replace(/</g, '&lt;').replace(/>/g, '&gt;').replace(/"/g, '&quot;');
   }
   ```
2. In the content building section, change:
   - `embed.originalUrl` -> `escapeHtml(embed.originalUrl)` in the `href` attribute
   - `embed.title` -> `escapeHtml(embed.title)` in the link text and item title
   - `post.body` -> `escapeHtml(post.body)` in the `<p>` tag content
   - Also escape `post.author_display_name` where used in `itemTitle`

**displayName sanitization (SEC-04):**

In `server/routes/auth.js`, sanitize displayName in BOTH registration routes (`/register` and `/register-reader`):

1. After the required-fields check, add sanitization:
   - Strip HTML tags: `displayName.replace(/<[^>]*>/g, '').trim()`
   - Limit to 50 characters: `.substring(0, 50)`
   - Reject if empty after stripping: return 400 `{ error: 'Display name is required' }`
2. Use the sanitized value for the rest of the handler (username generation, DB insert, response)

**Verify:** RSS feed content uses escaped HTML entities. Registration with `<script>alert('xss')</script>` as displayName stores stripped text.
</task>
