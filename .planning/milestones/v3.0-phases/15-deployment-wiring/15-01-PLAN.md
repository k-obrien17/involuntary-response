<task type="auto">
  <name>Task 1: Add admin env var validation to server startup</name>
  <files>server/index.js, server/db/index.js</files>
  <action>
  In server/index.js, BEFORE the `await initDatabase()` call, add a validation function that checks for required admin env vars. The function should:

  1. Check that ADMIN_EMAIL, ADMIN_PASSWORD, and ADMIN_DISPLAY_NAME are all set in process.env
  2. If ANY of the three are missing, throw an error with a clear message listing which vars are missing, e.g.: `"Missing required env vars: ADMIN_EMAIL, ADMIN_DISPLAY_NAME. Set these before starting the server."`
  3. The error should cause the process to exit (unhandled rejection or explicit process.exit(1))

  In server/db/index.js, update `seedAdmin()`:
  - Remove the early-return guard `if (!adminEmail || !adminPassword)` since validation now happens at startup
  - The function can assume all three vars exist (validated before initDatabase is called)
  - Keep the `ADMIN_DISPLAY_NAME` default fallback to 'Admin' as a safety net, but the startup validation should still require it

  Also in server/index.js, add a startup check for SMTP env vars:
  - If SMTP_HOST is not set, log: `"WARNING: SMTP_HOST not configured — password reset emails will be unavailable"`
  - This is a warning only, NOT a fatal error (server should still start)
  </action>
