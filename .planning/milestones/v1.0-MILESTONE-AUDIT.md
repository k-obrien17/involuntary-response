## Tech Debt (7 items across 4 categories)

### Deployment Config (pre-launch blocker)

1. **vercel.json missing `/api/*` proxy rewrite** — Browser API calls from Vercel frontend will be caught by SPA catch-all and receive HTML instead of JSON. All API-dependent functionality fails in production. Fix: add `{ "source": "/api/(.*)", "destination": "https://RENDER_URL/api/$1" }` before the SPA catch-all rewrite.

2. **RSS autodiscovery link uses relative URL** — `href="/api/feed"` on Vercel domain will be intercepted by catch-all. Resolved by the same API proxy rewrite above.

### Dead Code

3. **browse.byContributor()** in `client/src/api/client.js` — defined but never called. DISC-04 uses `profile.get()` instead.

4. **3 legacy Backyard Marquee route files** — `server/routes/artists.js`, `lineups.js`, `stats.js` exist but are not imported or mounted.

### Stale Documentation

5. **Phase 4 VERIFICATION.md** — Shows `status: gaps_found` but the gap was fixed. Should be updated to `status: passed`.

6. **Phase 2 VERIFICATION.md** — References `client/src/utils/embedParser.js` which was replaced by server-side `server/lib/oembed.js`. Architecture evolved correctly; docs are stale.

### Minor Code

7. **Silent truncation at 1200 chars** — `server/routes/posts.js` truncates body silently instead of rejecting with 400. Client enforces the limit, so this only affects direct API usage.
