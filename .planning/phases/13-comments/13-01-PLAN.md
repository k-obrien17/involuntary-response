**server/lib/post-helpers.js:**
1. In `batchLoadPostData`, add a `commentCountMap` object initialized at the top alongside the existing maps.
2. After the likedByUserMap logic, add a query:
   ```sql
   SELECT post_id, COUNT(*) as count FROM post_comments WHERE post_id IN (${ph}) GROUP BY post_id
   ```
   Loop results into `commentCountMap[row.post_id] = row.count`.
3. Add `commentCountMap` to the return object.
4. In `formatPosts`, add a 7th parameter `commentCountMap = {}` (backward-compatible default). Add `commentCount: commentCountMap[p.id] || 0` to the returned object.

**server/routes/posts.js:**
1. Add a `commentLimiter` rate limiter: `windowMs: 15 * 60 * 1000`, `max: 30`, message: `{ error: 'Too many comments, try again later' }`.

2. Add `POST /:slug/comments` endpoint (after the like endpoint, before GET /:slug):
   - Middleware: `authenticateToken, commentLimiter` (NOT requireContributor -- readers comment too)
   - Fetch post: `SELECT id FROM posts WHERE slug = ? AND status = 'published'` -- 404 if not found
   - Sanitize body: `sanitize(req.body.body, 500)` -- 400 if empty after sanitize
   - INSERT into post_comments (post_id, user_id, body)
   - SELECT the new comment with JOIN on users to get display_name, username, email
   - Return 201 with: `{ id, body, createdAt, author: { displayName, username, emailHash }, canDelete: true }`
   - canDelete is always true for the comment's own author

3. Add `DELETE /:slug/comments/:commentId` endpoint:
   - Middleware: `authenticateToken` only
   - Fetch post: `SELECT id, author_id FROM posts WHERE slug = ?` -- 404 if not found
   - Fetch comment: `SELECT id, user_id FROM post_comments WHERE id = ? AND post_id = ?` -- 404 if not found (validates comment belongs to post)
   - Three-way authorization check:
     - `isCommentAuthor = comment.user_id === req.user.id`
     - `isPostAuthor = post.author_id === req.user.id`
     - `isAdmin = req.user.role === 'admin'`
   - If none are true, return 403 `{ error: 'Not authorized to delete this comment' }`
   - DELETE FROM post_comments WHERE id = ?
   - Return `{ message: 'Comment deleted' }`

4. In the existing `GET /:slug` handler, after the like data section, add full comment loading:
   ```sql
   SELECT c.id, c.body, c.created_at, c.user_id,
          u.display_name, u.username, u.email
   FROM post_comments c
   JOIN users u ON c.user_id = u.id
   WHERE c.post_id = ?
   ORDER BY c.created_at ASC
   ```
   Map results to: `{ id, body, createdAt, author: { displayName, username, emailHash }, canDelete }` where canDelete = `(req.user && c.user_id === req.user.id) || (req.user && post.author_id === req.user.id) || (req.user && req.user.role === 'admin')`.
   Add `comments` array to the response JSON.

5. Update all `formatPosts` call sites in posts.js to pass `commentCountMap` as the 7th argument. The feed endpoint (GET /) already destructures from batchLoadPostData -- add `commentCountMap` to the destructured result. The call is:
   ```javascript
   const { embedMap, tagMap, artistMap, likeCountMap, likedByUserMap, commentCountMap } = await batchLoadPostData(postIds, req.user?.id);
   const posts = formatPosts(rows, embedMap, tagMap, artistMap, likeCountMap, likedByUserMap, commentCountMap);
   ```
