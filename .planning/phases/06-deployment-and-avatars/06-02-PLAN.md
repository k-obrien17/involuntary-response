Create a reusable `Avatar` component and integrate it into all views that display author information.

**1. Create `client/src/components/Avatar.jsx`:**

```jsx
/**
 * Avatar component — Gravatar with initials fallback.
 *
 * Props:
 *   emailHash (string) — MD5 hash of email for Gravatar lookup
 *   displayName (string) — Used for initials fallback
 *   size (number, default 32) — Pixel size (rendered as width/height)
 *   className (string, optional) — Additional CSS classes
 *
 * Strategy: Render initials on a colored background. Layer a Gravatar
 * image on top with d=blank (transparent fallback). If the user has
 * a Gravatar, it covers the initials. If not, the transparent image
 * reveals the initials underneath. No error handling or loading states needed.
 */
```

Implementation:
- Extract initials from `displayName`: take first character of first and last word, uppercase. If single word, use first character only.
- Generate a deterministic background color from the `emailHash` (take first 6 hex chars as color). Ensure text contrast by using white text.
- Gravatar URL: `https://gravatar.com/avatar/${emailHash}?d=blank&s=${size * 2}` (request 2x for retina).
- Render as a `<div>` with the initials centered, and an `<img>` absolutely positioned on top.
- Use `rounded-full` for circular avatar.
- The `<img>` should have `loading="lazy"` for performance in feed lists.

Size variants:
- `size={32}` — for PostCard and PostListItem (small, inline with text)
- `size={64}` — for Profile page (larger, standalone)
