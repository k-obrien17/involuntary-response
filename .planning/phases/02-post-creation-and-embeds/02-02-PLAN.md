**Step 2: Create client/src/utils/embedParser.js**

Create a new `utils/` directory in client/src/ if it does not exist.

Export a single function `parseEmbedUrl(url)` that:
- Returns null if url is falsy
- Matches Spotify URLs: `/https?:\/\/open\.spotify\.com\/(track|album|playlist)\/([a-zA-Z0-9]+)/`
  - Returns: `{ provider: 'spotify', type, id, embedUrl: 'https://open.spotify.com/embed/${type}/${id}', originalUrl: url }`
- Matches Apple Music URLs: `/https?:\/\/music\.apple\.com\/([a-z]{2})\/(album|playlist|song)\/[^/]+\/([0-9]+)(\?i=([0-9]+))?/`
  - If songId from `?i=` param: embedUrl = `https://embed.music.apple.com/${region}/${type}/x/${collectionId}?i=${songId}`, type = 'song'
  - Else: embedUrl = `https://embed.music.apple.com/${region}/${type}/x/${collectionId}`
  - Returns: `{ provider: 'apple', type, id: songId || collectionId, embedUrl, originalUrl: url }`
- Returns null if no match

**Step 4: Create client/src/components/EmbedPreview.jsx**

A component that renders a Spotify or Apple Music iframe based on embed data.

Props: `embed` (object with provider, type, embedUrl, title -- or null)

- If embed is null, render nothing.
- If provider === 'spotify': render iframe with src={embed.embedUrl}, width="100%", height={embed.type === 'track' ? 152 : 352}, frameBorder="0", allow="autoplay; clipboard-write; encrypted-media; fullscreen; picture-in-picture", loading="lazy", title={embed.title || 'Spotify embed'}. Wrap in a div with `className="rounded-lg overflow-hidden"`.
- If provider === 'apple': render iframe with src={embed.embedUrl}, width="100%", height={embed.type === 'song' ? 175 : 450}, frameBorder="0", sandbox="allow-forms allow-popups allow-same-origin allow-scripts allow-storage-access-by-user-activation allow-top-navigation-by-user-activation", allow="autoplay *; encrypted-media *; fullscreen *; picture-in-picture *", style={{ maxWidth: '660px' }}, title={embed.title || 'Apple Music embed'}. Wrap in a div with `className="rounded-lg overflow-hidden"`.
- Return null for unknown providers.

CRITICAL: Do NOT omit or simplify the `allow` or `sandbox` attributes. These are required for the embeds to function (see research pitfalls 1 and 2).
