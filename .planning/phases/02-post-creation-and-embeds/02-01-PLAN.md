```sql
-- posts table
CREATE TABLE IF NOT EXISTS posts (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  slug TEXT UNIQUE NOT NULL,
  body TEXT NOT NULL,
  author_id INTEGER NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (author_id) REFERENCES users(id)
);

-- post_embeds table (zero or one embed per post)
CREATE TABLE IF NOT EXISTS post_embeds (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  post_id INTEGER UNIQUE NOT NULL,
  provider TEXT NOT NULL,
  embed_type TEXT NOT NULL,
  embed_url TEXT NOT NULL,
  original_url TEXT NOT NULL,
  title TEXT,
  thumbnail_url TEXT,
  FOREIGN KEY (post_id) REFERENCES posts(id) ON DELETE CASCADE
);

-- post_tags table (max 5 per post, enforced in application)
CREATE TABLE IF NOT EXISTS post_tags (
  post_id INTEGER NOT NULL,
  tag TEXT NOT NULL,
  PRIMARY KEY (post_id, tag),
  FOREIGN KEY (post_id) REFERENCES posts(id) ON DELETE CASCADE
);

-- Indexes for future feed performance
CREATE INDEX IF NOT EXISTS idx_posts_created_at ON posts(created_at DESC);
CREATE INDEX IF NOT EXISTS idx_posts_author_id ON posts(author_id);
CREATE INDEX IF NOT EXISTS idx_post_tags_tag ON post_tags(tag);
```

Create server/routes/posts.js as a new Express Router module. Follow the patterns from the existing codebase (auth.js, invites.js) exactly: ESM imports, Router(), middleware chaining, parameterized SQL via db.execute, JSON responses.

**Imports:**
```javascript
import { Router } from 'express';
import { nanoid } from 'nanoid';
import rateLimit from 'express-rate-limit';
import { authenticateToken } from '../middleware/auth.js';
import db from '../db/index.js';
```

**Helper functions (defined at module scope):**

1. `sanitize(str, maxLength)` -- strips HTML tags, trims, slices to maxLength. Return null if input is null/undefined. Pattern: `String(str).trim().replace(/<[^>]*>/g, '').slice(0, maxLength)`.

2. `insertTags(postId, tags)` -- accepts post ID and array of tag strings. Process: slice to first 5, for each: lowercase, strip chars not matching `[a-z0-9\- ]`, trim, slice to 30 chars. Skip empty or duplicate (track with Set). Insert each via `INSERT OR IGNORE INTO post_tags (post_id, tag) VALUES (?, ?)`.

3. `parseEmbedUrl(url)` -- parses Spotify and Apple Music URLs. Returns `{ provider, type, id, embedUrl, originalUrl }` or null. Regex patterns:
   - Spotify: `/https?:\/\/open\.spotify\.com\/(track|album|playlist)\/([a-zA-Z0-9]+)/`
   - Apple Music: `/https?:\/\/music\.apple\.com\/([a-z]{2})\/(album|playlist|song)\/[^/]+\/([0-9]+)(\?i=([0-9]+))?/`
   - For Spotify: embedUrl = `https://open.spotify.com/embed/${type}/${id}`
   - For Apple Music: normalize region (use as-is), construct embedUrl = `https://embed.music.apple.com/${region}/${type}/x/${collectionId}` (append `?i=${songId}` if songId present from the `?i=` query param). Set type to 'song' if songId is present.

4. `fetchSpotifyMetadata(spotifyUrl)` -- calls `https://open.spotify.com/oembed?url=${encodeURIComponent(spotifyUrl)}` using native fetch. Returns `{ title, thumbnailUrl }` or null on any error. Non-fatal: embed works without metadata.

5. `validateEmbedDomain(embedUrl)` -- returns true only if embedUrl starts with `https://open.spotify.com/embed/` or `https://embed.music.apple.com/`. This is the server-side security check.

**Rate limiters:**
- `createLimiter`: 20 requests per 60 minutes, message: "Too many posts created, try again later"
- `updateLimiter`: 40 requests per 60 minutes, message: "Too many edits, try again later"

**Routes:**

**POST /** (create post) -- `authenticateToken, createLimiter`
1. Extract `body`, `embedUrl` (string), `tags` (array) from req.body.
2. Sanitize body with maxLength 1200. If empty after sanitize, return 400 "Post body is required".
3. Generate slug via `nanoid()`.
4. Insert into posts: `INSERT INTO posts (slug, body, author_id) VALUES (?, ?, ?)` with `[slug, body, req.user.id]`.
5. Get postId from `result.lastInsertRowid`.
6. If embedUrl provided: parse with parseEmbedUrl. If parsed is not null AND validateEmbedDomain(parsed.embedUrl) is true: fetch Spotify metadata if provider is 'spotify' (else metadata = {}). Insert into post_embeds: `(post_id, provider, embed_type, embed_url, original_url, title, thumbnail_url)`.
7. If tags provided: call insertTags(postId, tags).
8. Return 201 with `{ id: postId, slug }`.
9. Wrap in try/catch, return 500 on error.

**GET /:slug** (read single post)
No auth required (public). Query:
```sql
SELECT p.id, p.slug, p.body, p.author_id, p.created_at, p.updated_at,
       u.display_name as author_display_name, u.username as author_username
FROM posts p
JOIN users u ON p.author_id = u.id
WHERE p.slug = ?
```
If no row, return 404 "Post not found".
Then fetch embed: `SELECT provider, embed_type, embed_url, original_url, title, thumbnail_url FROM post_embeds WHERE post_id = ?`
Then fetch tags: `SELECT tag FROM post_tags WHERE post_id = ? ORDER BY tag`
Return JSON with post fields (camelCase: id, slug, body, authorId, createdAt, updatedAt, author: { displayName, username }), embed (as object or null), tags (as array of strings).

**PUT /:slug** (update post) -- `authenticateToken, updateLimiter`
1. Fetch post by slug. If not found, 404. If post.author_id !== req.user.id, 403.
2. Extract body, embedUrl, tags from req.body.
3. Sanitize body (maxLength 1200). If empty, 400.
4. Update posts: `UPDATE posts SET body = ?, updated_at = CURRENT_TIMESTAMP WHERE id = ?`.
5. Delete existing embed: `DELETE FROM post_embeds WHERE post_id = ?`.
6. If embedUrl provided: parse, validate domain, fetch metadata, insert new embed (same as create).
7. Delete existing tags: `DELETE FROM post_tags WHERE post_id = ?`.
8. If tags provided: insertTags(postId, tags).
9. Return 200 with `{ id: post.id, slug: post.slug }`.

**DELETE /:slug** (delete post) -- `authenticateToken`
1. Fetch post by slug. If not found, 404. If post.author_id !== req.user.id, 403.
2. Delete post: `DELETE FROM posts WHERE id = ?` (ON DELETE CASCADE handles embeds and tags).
3. Return 200 with `{ message: 'Post deleted' }`.

Export default router.

**Mount in server/index.js:**
1. Add import: `import postsRoutes from './routes/posts.js';`
2. Add route mount: `app.use('/api/posts', postsRoutes);` alongside the existing auth/invites/users mounts.
