<task type="auto">
  <name>Task 1: Add BigInt coercion to db.get and db.all</name>
  <files>server/db/index.js</files>
  <action>
  Add a `coerceRow` helper function above the `db` object definition in `server/db/index.js` that converts BigInt values to Number for all fields in a row object.

  1. Define the helper:
     ```javascript
     function coerceRow(row) {
       if (!row) return row;
       const out = {};
       for (const [key, value] of Object.entries(row)) {
         out[key] = typeof value === 'bigint' ? Number(value) : value;
       }
       return out;
     }
     ```
     Place this AFTER the `const client = createClient(...)` block and BEFORE the `const db = {` block.

  2. Update `db.get` to apply the coercion:
     ```javascript
     async get(sql, ...params) {
       const result = await client.execute({ sql, args: params });
       return coerceRow(result.rows[0]) || null;
     },
     ```
     Note: `coerceRow(undefined)` returns `undefined` which is falsy, so `|| null` still works correctly.

  3. Update `db.all` to apply the coercion to every row:
     ```javascript
     async all(sql, ...params) {
       const result = await client.execute({ sql, args: params });
       return result.rows.map(coerceRow);
     },
     ```

  Do NOT modify `db.run` or `db.exec` — they do not return row data.
  Do NOT modify anything else in the file (migrations, seedAdmin, initDatabase).

  WHY: libsql returns INTEGER columns as BigInt in JavaScript. The existing `db.run` coerces `lastInsertRowid` with `Number()`, but `db.get` and `db.all` pass raw rows through. This causes `user.id === post.authorId` to fail (1n !== 1) because user.id comes from JWT (Number) but post.author_id comes from a db query (BigInt). Coercing at the db wrapper layer fixes this globally for all queries without touching individual route handlers.
