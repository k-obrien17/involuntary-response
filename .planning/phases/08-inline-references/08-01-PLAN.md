<objective>
Parse Spotify and Apple Music URLs within post body text and render them as visually distinct styled links that open the streaming service, without embedding a player.

Purpose: Contributors often reference other songs or albums in their takes beyond the primary embed. Currently, if they paste a Spotify or Apple Music URL in the body text, it renders as an ugly raw URL string. This plan detects those music URLs and renders them as styled inline links -- visually distinct from surrounding text, clearly indicating the streaming service, and clickable to open the song/album.

Output: RichBody component that parses post text, PostCard and ViewPost updated to use it.
</objective>

  Create `client/src/components/RichBody.jsx` -- a component that takes post body text and renders it with Spotify and Apple Music URLs converted to styled inline links.

  1. Define the two music URL regex patterns at module scope:
     ```javascript
     const SPOTIFY_RE = /https?:\/\/open\.spotify\.com\/(track|album|playlist)\/[a-zA-Z0-9]+/g;
     const APPLE_MUSIC_RE = /https?:\/\/music\.apple\.com\/[a-z]{2}\/(album|playlist|song)\/[^/\s]+\/[0-9]+(\?i=[0-9]+)?/g;
     ```

  2. Create a combined regex that matches either pattern:
     ```javascript
     const MUSIC_URL_RE = new RegExp(`(${SPOTIFY_RE.source}|${APPLE_MUSIC_RE.source})`, 'g');
     ```

  3. Create a helper function `detectProvider(url)` that returns `'Spotify'` or `'Apple Music'` based on hostname.

  4. Create a helper function `detectType(url)` that returns `'track'`, `'album'`, or `'playlist'` (or `'song'` for Apple Music songs -- normalize to `'track'`).

  5. Build the `RichBody` component:
     ```jsx
     export default function RichBody({ text, className }) {
       // Split text on music URLs, preserving the URLs as separate segments
       // Use String.split with a capturing group regex to keep matches in the result array
       // For each segment:
       //   - If it matches a music URL: render as a styled <a> link
       //   - Otherwise: render as plain text
       // Wrap everything in a <p> with whitespace-pre-wrap and the provided className
     }
     ```

  6. For music URL segments, render as:
     ```jsx
     <a
       href={url}
       target="_blank"
       rel="noopener noreferrer"
       className="inline-flex items-center gap-1 text-current no-underline border-b border-gray-400 dark:border-gray-500 hover:border-gray-900 dark:hover:border-gray-200 transition"
     >
       <MusicIcon provider={provider} />
       <span>{provider} {type}</span>
     </a>
     ```

  7. Create a small inline `MusicIcon` component (or just an SVG element) that renders a small music note or service-appropriate icon:
     - For Spotify: A small green circle with the Spotify-style icon (simple: use a 🎵 text emoji or a minimal SVG). Actually, keep it minimal and text-first -- use a simple SVG music note icon (8x8 or 10x10) in the current text color, not brand-colored. This aligns with the site's minimal, text-first design.
     - Same icon for both providers (a small music note SVG). The provider name text distinguishes them.

  8. The link label should read like: "Spotify track", "Spotify album", "Apple Music track", "Apple Music album", etc. This is the human-readable label replacing the raw URL.

  9. Important implementation details:
     - Use `String.prototype.split()` with a capturing group regex: `text.split(/(https?:\/\/open\.spotify\.com\/...)/g)` -- this keeps the matched URLs in the result array at odd indices.
     - Actually, since we have two patterns, combine them into one capturing group for split: `text.split(MUSIC_URL_RE_CAPTURE)` where the outer regex has a single capturing group around the alternation.
     - Alternative cleaner approach: Use a loop with `MUSIC_URL_RE.exec()` or `matchAll()` to find all music URL positions, then build segments array of `{ type: 'text', value }` and `{ type: 'link', url, provider, mediaType }`, then map to JSX.
     - **Use the matchAll approach** -- it is more predictable and debuggable:
       ```javascript
       const parts = [];
       let lastIndex = 0;
       for (const match of text.matchAll(MUSIC_URL_RE)) {
         if (match.index > lastIndex) {
           parts.push({ type: 'text', value: text.slice(lastIndex, match.index) });
         }
         parts.push({ type: 'link', url: match[0], provider: detectProvider(match[0]), mediaType: detectType(match[0]) });
         lastIndex = match.index + match[0].length;
       }
       if (lastIndex < text.length) {
         parts.push({ type: 'text', value: text.slice(lastIndex) });
       }
       ```
     - If no music URLs found, `parts` will be `[{ type: 'text', value: text }]` -- renders identically to the current plain text.
     - Key each element by index since parts are derived from a stable string.

  10. The component should accept `className` prop and pass it to the wrapping `<p>` tag so PostCard and ViewPost can apply their existing typography classes.
