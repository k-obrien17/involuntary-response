API response shapes (from Plan 04-01):
```javascript
// GET /api/browse/tag/:tag
{ tag: "hip-hop", posts: [{ id, slug, body, createdAt, author: { displayName, username }, embed, tags, artists: [{ name, spotifyId }] }], nextCursor }

// GET /api/browse/artist/:name
{ artist: { name: "Kendrick Lamar", image: "https://..." }, posts: [...], nextCursor }
// Each post in posts[] includes artists array

// GET /api/browse/contributor/:username
{ contributor: { displayName, username }, posts: [...], nextCursor }
// Each post in posts[] includes artists array

// GET /api/browse/explore
{ tags: [{ tag, latest }], artists: [{ artist_name, artist_image, latest }], contributors: [{ username, display_name, latest }] }

// GET /api/users/:username/profile
{ user: { displayName, username, bio, createdAt }, posts: [...] }
// Each post in posts[] includes artists array: [{ name, spotifyId }]

// PUT /api/users/me
Request: { bio: "My bio text" }
Response: { bio: "My bio text" }
```

Note: The `artists` array on each post comes from the `post_artists` table (populated when a Spotify embed is added). It may be empty for posts without Spotify embeds. Artist names in this array are the entry point for the DISC-03 navigation path (artist name click -> artist page).
