- Export `getArtistsForSpotifyUrl(url)` — parses URL, calls Spotify Web API `/v1/tracks/{id}` or `/v1/albums/{id}`, returns array of `{ name, spotifyId }`. Returns empty array on failure (non-fatal). Uses `AbortSignal.timeout(5000)`. Returns empty array if SPOTIFY_CLIENT_ID or SPOTIFY_CLIENT_SECRET are missing.

**1. Create server/routes/browse.js** with four endpoints:

**GET /tag/:tag** — Posts filtered by tag:
- Lowercase the tag parameter
- Cursor-paginated (same composite cursor pattern as feed: `created_at|id`)
- Query: JOIN posts + users + post_tags WHERE pt.tag = ?, ORDER BY created_at DESC, id DESC
- Batch-load embeds and tags for returned posts using IN() (same pattern as feed in posts.js GET /)
- Return `{ tag, posts: [...], nextCursor }` where each post has: id, slug, body, createdAt, author: { displayName, username }, embed (object or null), tags (array)

**GET /artist/:name** — Posts featuring an artist:
- Use `decodeURIComponent(req.params.name)` for artist names with special chars
- Query post_artists WHERE artist_name = ? COLLATE NOCASE, JOIN posts + users
- For the artist header image: query `SELECT artist_image FROM post_artists WHERE artist_name = ? COLLATE NOCASE AND artist_image IS NOT NULL ORDER BY post_id DESC LIMIT 1`
- Cursor-paginated, batch-load embeds and tags
- Return `{ artist: { name, image }, posts: [...], nextCursor }`

**GET /contributor/:username** — Posts by a contributor (public data only):
- Query users WHERE username = ? AND is_active = 1 to get author info
- 404 if not found
- Query posts WHERE author_id = user.id, cursor-paginated, batch-load embeds and tags
- Return `{ contributor: { displayName, username }, posts: [...], nextCursor }`
- Note: This endpoint is separate from the profile endpoint — it provides the same compact post list format as tag/artist browse

**GET /explore** — Discovery hub:
- Three queries, each LIMIT 10:
  - Popular tags: `SELECT pt.tag, MAX(p.created_at) as latest FROM post_tags pt JOIN posts p ON pt.post_id = p.id GROUP BY pt.tag ORDER BY latest DESC LIMIT 10`
  - Top artists: `SELECT pa.artist_name, pa.artist_image, MAX(p.created_at) as latest FROM post_artists pa JOIN posts p ON pa.post_id = p.id GROUP BY pa.artist_name COLLATE NOCASE ORDER BY latest DESC LIMIT 10`
  - Active contributors: `SELECT u.username, u.display_name, MAX(p.created_at) as latest FROM users u JOIN posts p ON u.id = p.author_id WHERE u.is_active = 1 GROUP BY u.id ORDER BY latest DESC LIMIT 10`
- Return `{ tags: [...], artists: [...], contributors: [...] }`

**2. Create server/routes/profile.js** with two endpoints:

**GET /:username/profile** — Public profile page data:
- Query users WHERE username = ? AND is_active = 1, select id, display_name, username, bio, created_at
- 404 if not found
- Query posts WHERE author_id = user.id ORDER BY created_at DESC (no pagination needed for profiles — contributors write tens of posts, not thousands)
- Batch-load embeds and tags for the posts
- Return `{ user: { displayName, username, bio, createdAt }, posts: [...] }` where each post has same shape as feed posts

**PUT /me** — Update own bio (requires auth):
- Import and use `authenticateToken` middleware
- Extract bio from req.body.bio, sanitize: `String(bio).trim().replace(/<[^>]*>/g, '').slice(0, 300)`
- If bio is null/undefined, set bio to null (clear it)
- UPDATE users SET bio = ? WHERE id = req.user.id
- Return `{ bio }` (the sanitized value)

Ensure profileRoutes is mounted BEFORE usersRoutes in index.js so PUT /me is matched before any admin routes.

IMPORTANT: All browse endpoints use the same response shape for posts: `{ id, slug, body, createdAt, author: { displayName, username }, embed: {...} | null, tags: [...] }`. This enables a single PostListItem component on the frontend.
