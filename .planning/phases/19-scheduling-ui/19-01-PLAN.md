Extend PostForm to support scheduling:

2. Add state: `const [scheduledAt, setScheduledAt] = useState('')` — initialize from `initialScheduledAt` prop if provided (convert ISO UTC to datetime-local format: `new Date(initialScheduledAt).toISOString().slice(0, 16)` won't work for local time — instead use: `const d = new Date(initialScheduledAt); const local = new Date(d.getTime() - d.getTimezoneOffset() * 60000).toISOString().slice(0, 16);`).

3. Add a datetime-local input below the TagInput, inside a labeled section:
   - Label: "Schedule for later" (text-sm font-medium text-gray-700 dark:text-gray-300)
   - `<input type="datetime-local" value={scheduledAt} onChange={...} min={minDateTime} className="..." />`
   - `min` attribute: set to current time in local format (same conversion trick) to prevent picking past times
   - Helper text below: "Pick a date and time to auto-publish. Uses your local timezone." (text-xs text-gray-400)
   - Style the input: border border-gray-300 dark:border-gray-600 rounded-lg px-4 py-2 text-sm bg-white dark:bg-gray-800 dark:text-gray-100 focus:outline-none focus:ring-2 focus:ring-gray-900 dark:focus:ring-gray-400 mt-1

4. Add a "Schedule" button in the button row (between "Save as draft" and "Publish"):
   - Only show if `onSchedule` prop is provided AND `scheduledAt` is non-empty
   - Button text: "Schedule" (or "Reschedule" if `initialScheduledAt` was provided)
   - Style: blue-ish to distinguish from Publish — bg-blue-600 text-white px-6 py-3 text-sm font-medium rounded hover:bg-blue-700 transition disabled:opacity-50
   - onClick calls `onSchedule({ body, embedUrl, tags, artistName, scheduledAt: new Date(scheduledAt).toISOString() })` — converts local datetime-local value to UTC ISO string
   - Disabled when form is disabled (same isDisabled logic)

The key timezone handling: `<input type="datetime-local">` natively displays in the user's local timezone. When reading the value, `new Date(value)` interprets it as local time, then `.toISOString()` converts to UTC. This satisfies SCHED-02 with zero library overhead.

**CreatePost.jsx changes:**

1. Add `handleSchedule` handler (similar to handleSubmit):
   ```
   const handleSchedule = async ({ body, embedUrl, tags, artistName, scheduledAt }) => {
     setSubmitting(true);
     setError(null);
     try {
       const res = await posts.create({ body, embedUrl, tags, artistName, status: 'scheduled', scheduledAt });
       navigate('/my-posts');  // Go to My Posts so they can see it in the scheduled section
     } catch (err) {
       setError(err.response?.data?.error || 'Failed to schedule post');
       setSubmitting(false);
     }
   };
   ```

2. Pass `onSchedule={handleSchedule}` to PostForm.

**EditPost.jsx changes:**

1. The heading should also handle scheduled status: `post.status === 'draft' ? 'Edit draft' : post.status === 'scheduled' ? 'Edit scheduled post' : 'Edit post'`

2. Add `handleSchedule` handler for scheduling/rescheduling:
   ```
   const handleSchedule = async ({ body, embedUrl, tags, artistName, scheduledAt }) => {
     setSubmitting(true);
     setError(null);
     try {
       await posts.update(slug, { body, embedUrl, tags, artistName, status: 'scheduled', scheduledAt });
       navigate('/my-posts');
     } catch (err) {
       setError(err.response?.data?.error || 'Failed to schedule post');
       setSubmitting(false);
     }
   };
   ```

3. Pass `onSchedule={handleSchedule}` to PostForm (for both draft and scheduled posts — drafts can be scheduled, scheduled posts can be rescheduled).

4. Pass `initialScheduledAt={post.scheduledAt}` to PostForm when editing a scheduled post.

5. For scheduled posts, provide `onSaveDraft` to allow cancelling the schedule (revert to draft). The existing `handleSaveDraft` already calls `posts.update(slug, { body, embedUrl, tags, artistName })` without a status change — update it for scheduled posts to explicitly set `status: 'draft'`:
   ```
   const handleSaveDraft = async ({ body, embedUrl, tags, artistName }) => {
     setSubmitting(true);
     setError(null);
     try {
       const updateData = { body, embedUrl, tags, artistName };
       if (post.status === 'scheduled') {
         updateData.status = 'draft';
       }
       await posts.update(slug, updateData);
       navigate(post.status === 'scheduled' ? '/my-posts' : `/posts/${slug}`);
     } catch (err) {
       setError(err.response?.data?.error || 'Failed to save');
       setSubmitting(false);
     }
   };
   ```

6. Show `onSaveDraft` for both draft AND scheduled posts: change `onSaveDraft={post.status === 'draft' ? handleSaveDraft : undefined}` to `onSaveDraft={post.status === 'draft' || post.status === 'scheduled' ? handleSaveDraft : undefined}`

7. Do NOT show onSchedule for published posts (published->scheduled is rejected by API).

8. Update PostForm initialData to include `scheduledAt`: pass `initialScheduledAt={post.scheduledAt || null}` as a separate prop.
