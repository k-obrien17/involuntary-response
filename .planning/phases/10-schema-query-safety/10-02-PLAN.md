<objective>
Apply status filtering and cursor migration across all 5 route files that serve public post data. Refactor each file to use the shared post-helpers.js module, eliminating duplicated batch-load/format/cursor code.

Purpose: This is the safety audit -- the critical work that prevents unpublished content from leaking into any public surface. Every SQL query that returns posts to unauthenticated visitors must include `AND p.status = 'published'`, and every cursor must use `published_at` instead of `created_at`. The shared helper refactor ensures consistency and creates a single place for future like/comment count additions.

Output: All 5 route files updated with status filters, published_at cursors, and shared helper imports. GET /:slug supports author preview of drafts.
</objective>
