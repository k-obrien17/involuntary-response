  <name>Task 1: Create search API endpoint</name>
  <files>server/routes/search.js, server/index.js</files>
  <action>
Create `server/routes/search.js` with a `GET /` endpoint (mounted at `/api/search`).

**Query parameter:** `q` (required, string, trimmed). If missing or empty after trim, return 400 `{ error: 'Search query required' }`.

**Search logic:** Use a single SQL query that finds distinct posts matching the query across four dimensions. The query term should be wrapped with `%` wildcards for LIKE matching. Use `COLLATE NOCASE` for case-insensitive matching.

```sql
SELECT DISTINCT p.id, p.slug, p.body, p.created_at,
       u.display_name AS author_display_name, u.username AS author_username, u.email AS author_email
FROM posts p
JOIN users u ON p.author_id = u.id
LEFT JOIN post_artists pa ON pa.post_id = p.id
LEFT JOIN post_tags pt ON pt.post_id = p.id
WHERE (
  p.body LIKE ? COLLATE NOCASE
  OR pa.artist_name LIKE ? COLLATE NOCASE
  OR pt.tag LIKE ? COLLATE NOCASE
  OR u.display_name LIKE ? COLLATE NOCASE
  OR u.username LIKE ? COLLATE NOCASE
)
ORDER BY p.created_at DESC, p.id DESC
LIMIT ?
```

Pass the `%term%` pattern 5 times (once for each LIKE clause).

**Pagination:** Support cursor-based pagination matching the existing pattern from browse.js. Add `cursorClause`/`cursorParams` using the same `parseCursor` logic (copy the function). Default limit: 20, max 50.

**Post data assembly:** Copy `batchLoadPostData`, `formatPosts`, `emailHash`, and `parseCursor` from `browse.js` (these are not exported, so copy them). Apply `batchLoadPostData` to the result post IDs and `formatPosts` to build the response.

**Response shape:**
```json
{
  "query": "search term",
  "posts": [/* standard post shape */],
  "nextCursor": "2026-02-28T...|123" // or null
}
```

**Rate limit:** No special rate limit needed -- this is a read endpoint like browse.

**Mount in server/index.js:** Import `searchRoutes` from `./routes/search.js` and mount at `/api/search` (add line after the browse route mount: `app.use('/api/search', searchRoutes);`).
