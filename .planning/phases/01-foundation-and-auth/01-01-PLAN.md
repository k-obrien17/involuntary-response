**2. Replace server/routes/auth.js:**
Delete all existing BYM auth code. Create fresh with these endpoints:

**POST /api/auth/register** (rate limited — 20/15min):
- Body: `{ email, password, displayName, token }`
- Validate all 4 fields required, password >= 8 chars, email format basic check.
- Validate invite token: query `SELECT * FROM invite_tokens WHERE token = ? AND used_by IS NULL AND revoked_at IS NULL`. If not found, return 400 `{ error: 'Invalid or expired invite' }`.
- Check 7-day expiry in JavaScript: `if (Date.now() - new Date(invite.created_at + 'Z').getTime() > 7 * 24 * 60 * 60 * 1000)` return 400 expired. (Add 'Z' suffix for UTC per pitfall #2 from research.)
- Check email uniqueness: `SELECT 1 FROM users WHERE email = ?`. Return 409 if exists.
- Generate username from display name: lowercase, replace non-alphanumeric with hyphens, collapse multiple hyphens, trim hyphens, truncate to 20 chars, default to 'user'. Check uniqueness by appending -1, -2 etc. if collision.
- Hash password with bcrypt (10 rounds).
- INSERT user with role 'contributor'.
- Mark invite as used atomically: `UPDATE invite_tokens SET used_by = ?, used_at = CURRENT_TIMESTAMP WHERE token = ? AND used_by IS NULL AND revoked_at IS NULL` — check `result.changes === 0` for race condition (per pitfall #3 from research).
- Return 201 with `{ token: jwt, user: { id, email, displayName, username, role } }`.

**POST /api/auth/login** (rate limited — 20/15min):
- Body: `{ email, password }`
- Query user by email. If not found or is_active = 0, return 401 `{ error: 'Invalid email or password' }` (generic per user decision).
- bcrypt.compare. If fails, same 401 generic error.
- Return `{ token: jwt, user: { id, email, displayName: user.display_name, username, role } }`.

**POST /api/auth/forgot-password** (rate limited — 5/15min):
- Body: `{ email }`
- ALWAYS respond 200 `{ message: 'If an account exists, a reset link has been sent' }` immediately (prevents email enumeration).
- After response: look up user, if found and is_active, generate reset token (crypto.randomUUID()), store in password_reset_tokens with 1-hour expiry, call sendResetEmail.
- Wrap in try/catch — log errors but don't affect the already-sent response.

**POST /api/auth/reset-password**:
- Body: `{ token, password }`
- Validate password >= 8 chars.
- Query password_reset_tokens WHERE token = ? AND used_at IS NULL. Check expires_at not past.
- Hash new password, update user's password_hash.
- Mark reset token as used (used_at = CURRENT_TIMESTAMP).
- Return 200 `{ message: 'Password reset successful' }`.

Import `rateLimit` from express-rate-limit. Create authLimiter (20 req / 15 min) and forgotPasswordLimiter (5 req / 15 min).
