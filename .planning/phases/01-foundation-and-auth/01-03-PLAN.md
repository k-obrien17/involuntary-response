**1. Create server/routes/invites.js:**
All routes require authenticateToken + requireAdmin middleware.

**POST /api/invites** — Create invite:
- Body: `{ note }` (optional string, max 200 chars, trimmed).
- Generate token: `crypto.randomUUID()`.
- INSERT into invite_tokens (token, created_by = req.user.id, note).
- Return 201 with `{ id, token, note, createdAt, inviteUrl: FRONTEND_URL/register?token=UUID }`.
- Use `process.env.FRONTEND_URL` to construct the invite URL.

**GET /api/invites** — List all invites:
- Query all invite_tokens with JOIN to users for created_by display name and used_by display name.
- For each invite, compute status:
  - If revoked_at is not null: "revoked"
  - If used_by is not null: "used"
  - If created_at + 7 days < now: "expired"
  - Else: "pending"
- Return array of `{ id, token, note, status, createdBy: { id, displayName }, usedBy: { id, displayName, username } | null, createdAt, usedAt, revokedAt }`.
- Order by created_at DESC.

**PATCH /api/invites/:id/revoke** — Revoke unused invite:
- Validate invite exists and is not already used or revoked.
- UPDATE invite_tokens SET revoked_at = CURRENT_TIMESTAMP WHERE id = ? AND used_by IS NULL AND revoked_at IS NULL.
- Check result.changes — if 0, return 400 `{ error: 'Invite already used or revoked' }`.
- Return 200 `{ message: 'Invite revoked' }`.

**2. Replace server/routes/users.js:**
Delete old Backyard Marquee user profile routes. Create admin contributor management routes. All require authenticateToken + requireAdmin.

**GET /api/users/admin/contributors** — List all contributors:
- Query all users. Return array of `{ id, email, displayName: display_name, username, role, isActive: is_active, createdAt: created_at }`.
- Order by created_at DESC.

**PATCH /api/users/admin/:id/deactivate** — Deactivate contributor:
- Cannot deactivate yourself (check req.user.id !== param id).
- UPDATE users SET is_active = 0 WHERE id = ? AND id != req.user.id.
- Return 200 `{ message: 'Contributor deactivated' }`.

**PATCH /api/users/admin/:id/activate** — Reactivate contributor:
- UPDATE users SET is_active = 1 WHERE id = ?.
- Return 200 `{ message: 'Contributor activated' }`.

**PATCH /api/users/admin/:id/promote** — Promote to admin:
- Cannot demote yourself. Only promote (contributor -> admin).
- UPDATE users SET role = 'admin' WHERE id = ? AND role = 'contributor'.
- Check result.changes — if 0, return 400 `{ error: 'User is already an admin or not found' }`.
- Return 200 `{ message: 'Contributor promoted to admin' }`.
- Note in response: "Role change takes effect on their next login" (per pitfall #1 from research — stale JWT).

**3. Update server/index.js:**
- Add imports for invitesRoutes from './routes/invites.js' and usersRoutes from './routes/users.js' (the new admin users routes, replacing the old BYM import).
- Mount: `app.use('/api/invites', invitesRoutes)` and `app.use('/api/users', usersRoutes)`.
- Delete old BYM route imports that are still present (lineups, artists, stats) — these routes are for Backyard Marquee and not used in Involuntary Response. The route files themselves can remain on disk for reference but should NOT be imported.
